# Πώς να ανεβάσεις την εφαρμογή online (δωρεάν)

Στόχος: να πάρει η εφαρμογή σου ένα **πραγματικό URL** (π.χ. `https://sequence-toolkit.onrender.com`)
που μπορεί να ανοίξει οποιοσδήποτε από τον browser — όχι μόνο εσύ στον υπολογιστή σου.

Θα το κάνουμε με δύο δωρεάν εργαλεία: **GitHub** (αποθηκεύει τον κώδικα) και **Render**
(τρέχει τον κώδικα στο internet). Ο χρόνος: ~15 λεπτά, χωρίς κάρτα.

---

## Τι έχω ήδη ετοιμάσει για σένα

Μέσα στον φάκελο `sequence_webapp` υπάρχουν ήδη τα αρχεία που χρειάζεται το Render:

- **`render.yaml`** — λέει στο Render πώς να χτίσει και να τρέξει την εφαρμογή.
- **`runtime.txt`** — ορίζει την έκδοση Python (3.11).
- **`requirements.txt`** — οι βιβλιοθήκες.
- **`.gitignore`** — τι να ΜΗΝ ανέβει (προσωρινά αρχεία, η βάση).

Δεν χρειάζεται να τα αγγίξεις. Απλώς ακολούθησε τα βήματα.

---

## Βήμα 0 — Τι χρειάζεσαι μία φορά

- Λογαριασμό **GitHub** (δωρεάν): https://github.com/signup
- Λογαριασμό **Render** (δωρεάν): https://render.com — κάνε «Sign up with GitHub» (πιο εύκολο).
- Το **Git** εγκατεστημένο στον υπολογιστή σου: https://git-scm.com/download/win
  (Έλεγχος: άνοιξε PowerShell και γράψε `git --version`. Αν βγάλει αριθμό έκδοσης, το έχεις.)

---

## Βήμα 1 — Ανέβασε τον κώδικα στο GitHub

Άνοιξε PowerShell **μέσα στον φάκελο** `sequence_webapp`:

```powershell
cd C:\Users\perio\Desktop\Maggioli\sequence_webapp
git init
git add .
git commit -m "Sequence Toolkit web app"
```

Τώρα φτιάξε ένα άδειο repository στο GitHub:
1. Πήγαινε https://github.com/new
2. Στο «Repository name» γράψε `sequence-toolkit`.
3. Άφησέ το **Public**, μην τσεκάρεις τίποτα άλλο, πάτα **Create repository**.
4. Το GitHub θα σου δείξει εντολές. Χρησιμοποίησε αυτές (άλλαξε το `USERNAME` με το δικό σου):

```powershell
git remote add origin https://github.com/USERNAME/sequence-toolkit.git
git branch -M main
git push -u origin main
```

(Την πρώτη φορά θα σου ζητήσει να συνδεθείς στο GitHub — ακολούθησε το παράθυρο που ανοίγει.)

Μόλις τελειώσει, ανανέωσε τη σελίδα του repo στο GitHub — θα δεις όλα τα αρχεία σου εκεί.

---

## Βήμα 2 — Ανέβασέ το στο Render

1. Μπες στο https://dashboard.render.com
2. Πάνω δεξιά: **New +**  →  **Blueprint**.
3. Διάλεξε το repo `sequence-toolkit` (αν δεν φαίνεται, πάτα «Configure account» για να δώσεις πρόσβαση στο Render).
4. Το Render θα διαβάσει μόνο του το `render.yaml` και θα δείξει την υπηρεσία **sequence-toolkit**. Πάτα **Apply** / **Create**.
5. Ξεκινά το «build». Κράτησε υπομονή **5–10 λεπτά** την πρώτη φορά (εγκαθιστά scikit-learn κ.λπ.).

Όταν δεις πράσινο **«Live»**, η εφαρμογή είναι online.

---

## Βήμα 3 — Δες την εφαρμογή σου ζωντανά

Στην κορυφή της σελίδας του Render θα υπάρχει το URL, κάτι σαν:

```
https://sequence-toolkit.onrender.com
```

Άνοιξέ το σε οποιονδήποτε browser, σε οποιονδήποτε υπολογιστή ή κινητό. **Αυτό είναι το link
που δείχνεις στον υπεύθυνο.** Δοκίμασε και τα 4 tabs — δουλεύουν ακριβώς όπως τοπικά.

---

## Πώς κάνω αλλαγές μετά;

Κάθε φορά που αλλάζεις κάτι στον κώδικα, απλώς:

```powershell
git add .
git commit -m "τι άλλαξα"
git push
```

Το Render το καταλαβαίνει μόνο του και **ξαναχτίζει αυτόματα** την εφαρμογή. Δεν χρειάζεται
να κάνεις τίποτα άλλο.

---

## Καλό να ξέρεις (περιορισμοί δωρεάν πλάνου)

- **«Κοιμάται» μετά από αδράνεια.** Στο δωρεάν πλάνο, αν κανείς δεν μπει για ~15 λεπτά, η
  εφαρμογή αδρανεί. Η επόμενη επίσκεψη ξυπνά τον server και αργεί ~30 δευτερόλεπτα. Μετά
  τρέχει κανονικά. (Για τον υπεύθυνο, απλώς άνοιξέ το λίγο πριν του το δείξεις.)
- **Το ιστορικό μηδενίζει σε κάθε νέο deploy.** Η βάση `history.db` ζει στον προσωρινό δίσκο
  του Render, που καθαρίζει σε κάθε ανέβασμα. Για μόνιμο ιστορικό θα ήθελες Render Disk ή
  βάση Postgres — αλλά για επίδειξη δεν χρειάζεται.
- **NCBI:** το κατέβασμα από NCBI δουλεύει κανονικά online (ο server του Render έχει internet).

---

## Αν κάτι πάει στραβά

- **Το build απέτυχε:** άνοιξε την καρτέλα «Logs» στο Render και κοίτα την τελευταία κόκκινη
  γραμμή. Τις περισσότερες φορές φταίει μια βιβλιοθήκη — πες μου το μήνυμα και το φτιάχνουμε.
- **Βγαίνει «Not Found» στο άνοιγμα:** σιγουρέψου ότι το `render.yaml` ανέβηκε στο GitHub
  (φαίνεται στη λίστα αρχείων του repo).
- **Το `git push` ζητάει password και δεν δέχεται:** το GitHub θέλει «Personal Access Token»
  αντί για κωδικό. Ο ευκολότερος τρόπος: εγκατέστησε το **GitHub Desktop** και κάνε push από εκεί
  με κουμπιά.

Μόλις γίνει Live, στείλε μου το URL να το δούμε μαζί.
